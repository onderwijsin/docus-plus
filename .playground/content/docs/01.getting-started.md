This playground is a minimal example of a project built on top of Docus Plus. The layer provides
the documentation shell; the project supplies its content, identity, visual tokens, and brand
components.

## Add your first page

Create a Markdown file below `content/docs/`. Nuxt Content turns the file into a route and Docus
uses its frontmatter to build the navigation.

```md [content/docs/01.getting-started.md]
---
title: Getting Started
description: Create a small documentation site.
---

Your guide starts here.
```

Use numbered filenames when you want to control the order of pages in a section. Add a
`.navigation.yml` file when you want to give a section a title or icon.

## Configure the consuming app

The playground's Nuxt config extends the local layer. A published project uses a versioned GitHub
reference instead:

```ts [nuxt.config.ts]
export default defineNuxtConfig({
  extends: ["github:onderwijsin/docus-plus@v0.0.1"]
});
```

The project can then add `app/app.config.ts`, `app/app.css`, and component overrides without
editing the layer itself.

::tip{title="Use the playground as a reference"}
The landing page, app configuration, stylesheet, and `AppHeaderLogo` override in this directory
show the smallest useful consumer setup.
::
