The layer includes optional integrations for AI assistance, API reference pages, health checks,
cache storage, and Cloudflare deployment. The consumer enables and configures them through its
Nuxt config and environment.

## Configure the assistant

This starter provides an assistant endpoint and uses the `ai` configuration from the layer. A
consumer can override the model and behavior without changing shared server code:

```ts [nuxt.config.ts]
export default defineNuxtConfig({
  ai: {
    model: "mistral-large-latest",
    maxOutputTokens: 12000,
    maxRetries: 3,
    maxSteps: 12,
    temperature: 0.2,
    systemPrompt: "You are the documentation assistant for Example Docs."
  }
});
```

The `systemPrompt` replaces the generated documentation-aware prompt. Supply provider API keys
through the environment rather than committing them to the repository.

## Add API reference content

When OpenAPI settings are provided, the layer can create API content from a local or remote OpenAPI
document. The starter selects its API document through `config/openapi.ts` and exposes it at the
configured Scalar route.

For a consuming project, configure the source explicitly:

```dotenv
OPENAPI_SOURCE_TYPE=remote
OPENAPI_LOCATION=https://example.com/openapi.json
```

Keep API source selection site-specific. The reusable layer only provides the collection and module
integration.

See [Scalar and OpenAPI reference](/guides/scalar-and-openapi) for how the document is parsed and
how the generated API content is used by search and the assistant.

## Configure site identity

Set the public URL and site identity in the consumer. The URL is used for canonical links, sitemap
entries, and absolute social image URLs:

```ts [nuxt.config.ts]
export default defineNuxtConfig({
  site: {
    url: "https://docs.example.com",
    name: "Example Docs",
    description: "Documentation for the Example platform."
  }
});
```

In the deployment environment, set the corresponding public URL:

```dotenv
NUXT_PUBLIC_SITE_URL=https://docs.example.com
```

## Build for production

Docus Plus requires server routes for the assistant, health checks, API reference, and runtime OG
images. Build the application as a server deployment:

```bash
pnpm build
```

The application does not support `pnpm generate` or zero-runtime OG images. Deploy the
resulting Nuxt server using the target's normal runtime configuration.

For OG image requirements, rendering, and template overrides, see
[OG image rendering](/guides/og-images).

For the recommended environment setup with Varlock and Proton Pass, see
[Manage environment variables with Varlock](/guides/varlock-and-proton-pass).

::tip{title="Keep deployment configuration in the consumer"}
The layer supports the deployment paths, but the consuming project owns its public URL, secrets,
Cloudflare bindings, cache policy, and final hosting setup.
::
